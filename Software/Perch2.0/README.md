# Perch 2.0 - Anomaly Sound Detection

This directory contains the Perch 2.0 version of our Anomaly Sound Detection project. The pipeline leverages acoustic embeddings (from the Perch model) to identify abnormal audio events by comparing them against a statistically defined "normal" reference profile.

## Directory Structure Overview

### 1. `Code/`
This folder contains all the Python scripts required to process the audio, run the anomaly detection models, and evaluate the results.

**Key Scripts & Functions:**

- **`Treatment.py`**: The main data pipeline script.
  - `frac_audio()` splits long continuous recordings into standardized 5-second segments.
  - `compute_and_save_png()` generates visual Mel-Spectrograms from audio snippets.
  - `extract_for_validation()` grabs random mixes of anomalous and normal audio snippets into a dedicated folder and creates a dummy CSV for humans to later validate manually.
  - `amplify_directory()` applies digital gain to amplify low-volume audio segments.

- **`Anomaly_detector_gaussian.py`**: The core detection algorithm.
  - `get_audio_signature()` runs audio through an ONNX Perch model to extract compact embeddings.
  - `assess_neutral()` and `compile_reference()` analyze a folder full of "neutral" audio and build a mathematical reference (mean and standard deviation) to define normality.
  - `anomaly_detector_gaussian()` is the main routine. It loops over target sets, measures the inverse-variance weighted Cosine distance against the reference. Very close samples automatically enrich the neutral folder, while distant samples are marked as Anomalies and written to a CSV tracking report.

- **`interface_validation_perch.py`**: A complete desktop GUI (PerchExplorer) for manual annotation.
  - Built with `tkinter`, `pygame`, and `matplotlib`.
  - It loads the generated anomaly CSV, allows humans to visually inspect the Mel-Spectrogram, play the 5-second snippet, and physically validate if it is an `Anomaly` (!), `Void/Normal` (✓), or `uncertain` (?).
  - Automatically saves the votes and tracks annotation progress.

- **`F1-score.py`**: The metrics and optimization script.
  - `find_optimal_threshold()` compares the machine's distance scores to the human truth annotations produced by the PerchExplorer tool.
  - It computes the Precision-Recall curve to find the exact anomaly threshold distance that maximizes the dataset's F1-Score and plots a curve.

### 2. `reference/`
Stores the mathematically established "normal" state. These `.npy` files pre-compute the statistics of normal conditions to speed up inference:
- **`reference_signature_optimize.npy`**: The mean feature vector (centroid) of the normal audio embeddings.
- **`reference_std_optimize.npy`**: The standard deviation/covariance metrics of the normal audio. Used directly by the Gaussian anomaly detector.

### 3. `Result/`
Contains all the output artifacts generated by the code, including CSV files with scores and data visualizations.
- **`Anatole_validation.csv`**: Ground truth annotations or validation dataset used to check the accuracy of the system.
- **`cosine_final_synced.csv`**: Outputs and anomaly scores calculated using a Cosine Similarity approach.
- **`tree_final_synced.csv`**: Additional synchronized result outputs.
- **`gaussian_similarity_PC8.png`**: A graph plotting the Gaussian similarity distribution, useful for determining the anomaly threshold.
- **`SAMPLES_EXCTRACTED.png`**: A visualisation of sperm whale clicks.
